Cduino Prerequisites

Required Hardware

  • Arduino Duemilanove or Arduino Uno with USB programming cable. If you use an older duemilanove, some variable settings may need to be tweaked away from the defaults (which assume a Mega328P chip).
  • If you want to use in-system programming (ISP), an AVRISPmkII or clone is needed. Probably any programmer supported by avrdude could be made to work with a trivial tweak or two to generic.mk.
  • Many individual modules require additional hardware for testing. These requirements are documented at the top of the *_test.c test program for the module in question.

Required Software

  • GNU AVR GCC compiler (debian package: gcc-avr)
  • GNU AVR binutils (debian package: binutils-avr)
  • GNU AVR libc (debian package: avr-libc)
  • avrdude (debian package: avrdude)
  • screen for terminal-style communication with the arduino (debian package: screen)

Recommended Software

  • libusb (debian package: libusb-1.0-0-dev ) is required if you want to build avrdude with suppor for use of an AVRISPmkII (or other in-system programmer) for programming. There's a good chance your system already includes this library by default, and any pre-packaged version of avrdude probably includes it.
  • Device::SerialPort perl module (debian package: libdevice-serialport-perl ) If present, this module will be used to reset the Arduino when you program it with the Arduino bootloader, saving you the trouble of pushing the reset button all the time.
